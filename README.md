# Library Management System

## Project Information

### Project Name

Library Management System

### Team Members

* Mohanad khaled mohamed 

### Project Description

The Library Management System is a web-based application that helps users search for books, borrow books, and return borrowed books. The system provides a simple interface for managing library resources.

## Software Classification

### Software Type

Information Management System

### Product Type

Generic Product

## Initial Quality Analysis

### 1. Usability

The system should be easy to use so users can quickly search, borrow, and return books without confusion.

### 2. Reliability

The system must correctly update book availability and borrowing status to avoid incorrect information.

### 3. Security

User login information should be protected from unauthorized access.

## Possible Software Risk

Unauthorized access to the system may allow users to modify library information or view restricted data.

## AI Development Process

### AI Tool Used

ChatGPT

### Final Prompt

Create a Minimum Viable Product (MVP) for a Library Management System using HTML, CSS, and JavaScript.

Requirements:

* Login page for users
* Dashboard displaying available books
* Search books by title
* Borrow a book
* Return a book
* Display borrowed status for each book
* Responsive and user-friendly interface
* Use sample book data stored in JavaScript
* Organize the code into separate HTML, CSS, and JS files
* Include comments explaining the code

### Prompt Improvement

The first prompt requested only the basic functionality. The second prompt added detailed requirements such as responsiveness, code organization, comments, and book status management, which resulted in a more complete MVP.
